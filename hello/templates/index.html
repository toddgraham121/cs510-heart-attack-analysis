Cardiac Disease (description)

  • This app was deployed to Heroku, either using Git or by using Heroku Button on the repository.
  • When Heroku received the source code, it fetched all the dependencies in requirements.txt, creating a deployable slug.
  • The platform then spins up a dyno, a lightweight container that provides an isolated environment in which the slug can be mounted and executed.
  • You can scale your app, manage it, and deploy over 150 add-on services, from the Dashboard or CLI.

Description of Dataset

  • This dataset is a combination of four databases and it consists of a total of 1025 patients where 312 are women and 713 are men.
  • There are a total of 14 attribures, but we will be using just 5 for the dashboard and 8 for the classifier.
